# üé® DesignAssets

A clean, lightweight Swift Package Manager package for consuming design assets from Figma with automatic build-time code generation and type-safe access.

## ‚ú® Features

- **üì¶ Clean SPM Package**: Lightweight package focused on iOS asset consumption
- **üîå BuildToolPlugin**: Automatic Swift code generation at build time

- **‚ö° Build-Time Generation**: Generates Swift enums automatically during compilation
- **üì± SwiftUI & UIKit**: Works with both SwiftUI and UIKit
- **ü§ñ External Workflow**: Icons are fetched and updated via FigmaIconsWorkflow repository
- **üåê Cross-Platform Ready**: Same assets can be consumed by any platform
- **üîÑ Zero Configuration**: Just add the package and build
- **‚ö° Type-Safe Access**: Compile-time checking for all icons
- **üöÄ Remote Package Compatible**: Works seamlessly when added remotely from GitHub

## ü§ñ Workflow Setup

### **FigmaIconsWorkflow Repository**

The icon fetching and management is handled by a separate repository:

- **Repository**: [FigmaIconsWorkflow](https://github.com/babladiabhishek/FigmaIconsWorkflow)
- **Purpose**: Monitors Figma files and creates pull requests with updated icons
- **Benefits**: Keeps the SPM package lightweight and focused on consumption

### **Required Secrets (in FigmaIconsWorkflow)**

Add these secrets to the FigmaIconsWorkflow repository:

1. **`FIGMA_PERSONAL_TOKEN`**
   - Go to [Figma Settings](https://www.figma.com/settings)
   - Navigate to **Account** ‚Üí **Personal Access Tokens**
   - Create a new token with file access permissions
   - Add it as a repository secret

2. **`FIGMA_FILE_ID`**
   - Extract from your Figma file URL
   - Example: `https://www.figma.com/design/T0ahWzB1fWx5BojSMkfiAE/Icons`
   - File ID: `T0ahWzB1fWx5BojSMkfiAE`
   - Add it as a repository secret

3. **`PAT_TOKEN`**
   - GitHub Personal Access Token with repository access
   - Used to create pull requests in this DesignAssets repository

### **Workflow Benefits**

- **‚ö° Ultra-Fast**: Saves 2-3 minutes on unchanged files
- **üîÑ Smart Caching**: Only fetches when Figma actually changes
- **üöÄ Swift 5.8**: Compatible Swift toolchain for optimal performance
- **üìä Change Detection**: Uses Figma's `lastModified` timestamp
- **üéØ Force Override**: Manual refresh when needed
- **ü§ñ Automation**: No manual icon management needed
- **üåê Cross-Platform**: Same assets can be consumed by any platform
- **üìù Version Control**: Track icon changes in Git history

## üîß Troubleshooting

### **Common Issues**

1. **"Invalid token" error**
   - Make sure your Figma token has "File" permissions
   - Check that the token is correctly formatted

2. **"Token expired" error**
   - Generate a new token from Figma settings
   - Update your environment variables

3. **"No icons found"**
   - Verify the file ID is correct
   - Check that the Figma file contains icon components
   - Ensure your token has access to the file

4. **"No changes detected" (workflow skipping fetch)**
   - This is normal behavior when Figma hasn't changed
   - Use `--force` flag or force refresh in manual workflow to override

5. **"Swift 6.2 not available" error**
   - The workflow uses the latest Swift toolchain
   - Check GitHub Actions logs for specific version issues

6. **Permission errors**
   - Ensure environment variables are set correctly
   - Check that the Python script has write permissions

7. **"git-lfs: command not found" error (RESOLVED)**
   - ‚úÖ **Fixed**: All SVG files are now stored directly in Git
   - ‚úÖ **No Git LFS required**: Package works without Git LFS installation
   - ‚úÖ **Remote compatibility**: Works seamlessly when added remotely

8. **"public headers include directory" error (RESOLVED)**
   - ‚úÖ **Fixed**: Package.swift updated for better remote resolution
   - ‚úÖ **Explicit paths**: All targets have explicit paths for compatibility
   - ‚úÖ **Platform compatibility**: Lowered requirements to iOS 13, macOS 10.15
